# Release Notes Template

发布新版本时，建议按下面结构填写 GitHub Release 说明。

## 新增

- 新增功能 1
- 新增功能 2

## 修复

- 修复问题 1
- 修复问题 2

## 升级说明

- 是否需要用户手动操作，例如重新登录、重新授权、重启应用等。
- 如果是 Android 安装包更新，首次安装或应用内升级时，系统可能要求允许“安装未知来源应用”或“允许来自此来源的应用”。
- 当前发布包文件：`your-app-x.y.z+build.apk`
- SHA256：`替换为实际摘要`

## 已知问题

- 如无可删除本节。
- 如有已知限制或兼容性说明，可写在这里。

## 备注

- 面向用户的版本说明尽量简洁，优先写用户能感知到的变化。
- 如果某次版本主要是内部修复，也建议至少说明修复范围，避免版本记录失去意义。
